XP Pickup Attraction

Physics-driven XP pickups that settle naturally on the ground but stay responsive enough to snap toward the player once they're in range.

Blueprints Systems
screenshot / clip

The Problem

Pickups used a Projectile Movement Component, and stopping simulation on ground impact (to keep them from sliding forever) also killed the physics response needed for the later "fly toward the player" attraction behavior.

Approach

  • Kept bShouldBounce set to true instead of calling a hard stop on ground impact.
  • Paired that with low bounciness and high friction, so pickups still settle quickly and don't visibly bounce around, without disabling simulation.
  • This kept the Projectile Movement Component "live," so redirecting velocity toward the player once the attraction radius triggers still works cleanly.

Breakdown

// what didn't work On ground impact → StopSimulating() // kills physics response entirely // what worked instead bShouldBounce = true Bounciness = low Friction = high // pickup settles fast, but stays physically responsive

Outcome

Pickups feel alive at rest and still snap smoothly toward the player when the attraction radius triggers, without needing a separate movement-mode swap.
