Outline Effect Component

A drop-in component that gives any actor a selectable-object outline, without per-actor setup.

Blueprints
screenshot / clip

The Problem

Highlighting interactable or selected objects needed to work consistently across very different actor types — props, NPCs, pickups — without duplicating outline logic on each one.

Approach

  • Built BP_OutlineComponent around Custom Depth Stencil IDs, assigning each highlighted actor a stencil value.
  • Paired it with a post-process material that reads the stencil buffer and draws an outline along the edges of any actor carrying a matching ID.
  • Packaged the whole thing as a single component, so enabling the effect on a new actor is just "add the component, set a stencil ID" — no per-actor material or render logic.

Breakdown

// usage Add BP_OutlineComponent to Actor Set StencilID = n // under the hood Actor.CustomDepthStencilValue = StencilID PostProcessMaterial reads stencil buffer → draws edge outline

Outcome

Any actor can get a consistent, art-directable highlight by adding one component — no bespoke outline setup needed per actor type.
