Weight-Based Fish Scaling

A formula-driven system that scales a caught fish's mesh to match its rolled weight, instead of relying on a fixed set of pre-scaled meshes.

Blueprints Systems
screenshot / clip

The Problem

A handful of fixed mesh sizes per fish species looks repetitive and breaks the illusion that weight is meaningful — but scaling linearly with weight makes big fish look absurdly oversized, since volume grows with the cube of linear size, not weight directly.

Approach

  • Used a cube-root relationship between weight and scale, since weight scales roughly with volume (length³) rather than length — this keeps big fish looking proportionally right instead of ballooning.
  • Clamped the resulting scale at both ends so extreme weight rolls can't produce a mesh that's too small to read or large enough to look glitchy.
  • Ran the raw formula output through a Curve Float asset before applying it, so art direction can hand-tune the falloff without touching the underlying math.
  • Applied non-uniform scaling so girth can be nudged independently of overall length, avoiding the "stretched balloon" look on heavier fish.

Breakdown

// Blueprint node flow RawScale = Power( Weight / ReferenceWeight, 1 / 3 ) FinalScale = Clamp( RawScale, MinScale, MaxScale ) FinalScale = CurveFloat.GetValue( FinalScale ) // art-directed remap Apply FinalScale to mesh (non-uniform for girth)

Outcome

Fish read as believably sized across the whole weight range, and artists can reshape the size curve directly in a Curve Float asset without needing a single line of new logic.
