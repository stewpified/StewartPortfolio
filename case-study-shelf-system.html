Shelf Placement & Sorting

A slot-based system for placing and tracking items on shelves, built around a map of slot keys to occupying items.

Blueprints Systems
screenshot / clip

The Problem

Slots would occasionally get stuck as permanently occupied even after the item on them was removed, making them unusable for the rest of the session.

Approach

  • Traced the bug to how slots were being freed: calling Remove() on the TMap deleted the slot key entirely, rather than just clearing its value — so later lookups for that slot key failed silently instead of reporting it as empty.
  • Fixed it by clearing a slot's occupant with Map.Add(SlotKey, None) instead of removing the key, keeping every slot key present in the map at all times.
  • Explored refactoring toward a TArray of structs (each carrying a SlotTransform and an OccupyingBox reference) for clearer per-slot ownership and easier iteration than a parallel TMap.

Breakdown

// the bug Slots.Remove(SlotKey) // deletes the key — later lookups fail // the fix Slots.Add(SlotKey, None) // key stays, value cleared // refactor direction Struct FShelfSlot SlotTransform OccupyingBox // null when empty

Outcome

Eliminated the ghost-slot bug entirely, and the struct-array direction makes it straightforward to iterate every shelf slot for restocking or sorting logic later.
